쉘과 터미널의 차이

쉘: 명령어 반응형 프로그램
터미널: 입, 출력 환경

------------------------

리눅스 디렉토리 구조

기초 디렉토리
/bin : binary의 줄임말 
        기본 명령어(프로그램) 전용 디렉토리

/sbin : system binary의 줄임말 
        시스템 관리자 권한이 필요한 프로그램들이 위치
        시스템 컨트롤 명령어 전용 디렉토리

/etc : 설정 파일 전용 디렉토리

/lib : 라이브러리 파일 전용 디렉토리

유저를 위한 디렉토리
/home : 사용자 계정별 전용 디렉토리

/root : root 사용자 계정별 전용 디렉토리

/usr : 배보판 마다 달라지는 프로그램 디렉토리
/usr/bin : 일반 프로그램 설치 기본 경로
/usr/sbin: 시스템 명령어 프로그램 설치 기본 경로
/usr/lib: 라이브러리 및 커널모듈 설치 기본 경로
/usr/include : C 프로그램 개발시 필요한 헤더파일 전용
/usr/local : 소스코드 컴파일 후 설치 기본 경로

/opt : 추가 프로그램 설치시 활요하는 경로

시스템 디렉토리 

/sys : 리눅스 시스템(커널) 기능 및 데이터 가상파일들 (디스크에 저장되지 않고 커널 메모리에 존재)

/proc : 프로세스의 정보 또는 시스템의 정보 조회 가상파일 (디스크에 저장되지 않고 커널 메모리에 존재)

/boot : 리눅스 부팅시 필요한 파일 전용 폴더
    vmlinuz: C 리눅스 바이너리
        vm : 가상메모리 사용
         z : 압축

/dev : 연결된 디아비스를 다루는 가상 파일들 (디스크에 저장되지 않고 커널 메모리에 존재)
    sda : SCSI Disk a

/var : 각종 로그 및 데이터를 저장하는 경로
    로그 로테이트: 로그 파일을 분할 압축하여 저장되지

/run : 프로그램 실행도중에 쓰이는 runtime 파일 전용

다용도 디렉토리 및 복구 디렉토리

/tmp : 임시파일 저장 용도 (systemd 등 활용)

/mnt, /media : 새로운 디스크(USB등) 연결 전용 디렉토리

/lost+found : fsck 등 파일시스템 체크 / 복구 파일 전용
    복구되지않은 채로 블록만 존재하는 파일 -> 연결이 끊어진 inode 들이 숫자파일 형태로 존재하는 곳
    숫자 형태로 존재하는 파일들은 mv 명령어로 파일 이름만 바꾸면 바로 복구될수 있다
